# DoMu — музыкальный сайт

Полноценный сайт: главное меню, авторизация (регистрация/вход), список треков и нижний плеер (play/пауза/след/назад).

## Важно про аудио
В демо используются реальные названия и имена исполнителей (Billie Eilish, The Weeknd и т.д.), но сам звук — это открытые бесплатные демо-дорожки (SoundHelix), потому что настоящие записи защищены авторским правом и я не могу их встраивать без лицензии.

Если хочешь реальный звук, варианты:
- Загрузить собственные лицензированные mp3-файлы и заменить `audioUrl` в `data/db.json` на путь к своим файлам (например `/audio/track1.mp3`, положив файлы в `public/audio/`).
- Подключить официальный embed Spotify (iframe) — это законно, звук идёт с серверов Spotify.
- Использовать YouTube API/embed для тех же целей.

## Запуск локально

Понадобится [Node.js](https://nodejs.org) (версия 18+).

```bash
cd domu
npm install
npm start
```

Открой в браузере: http://localhost:3000

## Структура проекта

```
domu/
  server.js        — Express-сервер, API
  package.json
  data/db.json      — "база данных" (пользователи + треки), простой JSON-файл
  public/
    index.html      — весь фронтенд (дизайн + разметка)
    app.js          — логика: плеер, авторизация, рендер треков
```

## API

| Метод | Путь | Описание |
|---|---|---|
| GET | /api/tracks | список треков |
| POST | /api/auth/register | { username, password } → { token } |
| POST | /api/auth/login | { username, password } → { token } |
| GET | /api/me | текущий пользователь (нужен токен) |

## Как выложить сайт в интернет бесплатно (Render.com)

Самый простой и надёжный бесплатный способ для Node.js-сайта — **Render.com**.

### Шаг 1. Залей проект на GitHub
1. Зарегистрируйся на [github.com](https://github.com), если ещё нет аккаунта.
2. Создай новый репозиторий (например `domu-site`).
3. Загрузи туда всю папку `domu` (можно через сайт GitHub — кнопка "uploading an existing file", просто перетащи все файлы, кроме `node_modules`, если он появится).

### Шаг 2. Разверни на Render
1. Зайди на [render.com](https://render.com) и зарегистрируйся (можно через GitHub-аккаунт).
2. Нажми **New +** → **Web Service**.
3. Подключи свой GitHub-репозиторий `domu-site`.
4. В настройках укажи:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. Нажми **Create Web Service**.
6. Через 1–2 минуты Render даст тебе ссылку вида `https://domu-site.onrender.com` — это уже рабочий сайт в интернете.

### Особенности бесплатного тарифа Render
- Сайт "засыпает" после ~15 минут без посетителей и просыпается за 20–30 секунд при следующем заходе — это нормально для бесплатного плана.
- Если это критично, можно настроить бесплатный пинг-сервис (например uptimerobot.com), который будет заходить на сайт каждые 10 минут, чтобы он не засыпал.

### Альтернатива — Glitch.com (ещё проще, без GitHub)
1. Зайди на [glitch.com](https://glitch.com) → **New Project** → **Import from GitHub** (или создай пустой Node-проект и вручную вставь файлы).
2. Glitch сам установит зависимости и запустит `npm start`.
3. Сайт сразу доступен по ссылке вида `https://твой-проект.glitch.me`.

Оба варианта — полностью бесплатные, без карты, без "пробного периода".
